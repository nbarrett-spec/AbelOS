## Executive Summary

The PM-facing tools in Aegis are **structurally strong but operationally incomplete**. The PM Command Center, Job Pipeline, Schedule & Dispatch, Crews, Inspections, and Lien Releases pages all load and function. The job detail page is the richest page in the system — 12-stage progress tracker, readiness checklist, change orders, punch list, delivery sign-off, allocation ledger, and document sync tabs.

**However, a PM cannot complete a full house lifecycle in the system today.** Critical gaps: no order detail page (404), Material Calendar empty despite 367 active jobs, Assign Installer broken (schema error exposed to user), Hyphen sync returning 401, no role-filtered views, and $0.00 pricing across all AI-recommended POs.

**Severity breakdown:** 5 blockers, 8 major issues, 8 enhancement opportunities.

## Findings — Bugs & Broken Features

### BLOCKER-1: No Order Detail Page (404)
- **Location:** `/ops/orders/{SO-number}`
- **Impact:** PM cannot view line items, delivery info, payment status, or any detail for any of the 4,534 orders. The Orders list shows order numbers but rows aren't clickable and there's no detail route.
- **Fix:** Build order detail page or wire rows to navigate to it. The job detail page already links to `SO-003223 →` — that link target needs to exist.

### BLOCKER-2: Assign Installer — Schema Error Exposed
- **Location:** Job detail page → right sidebar → "Assign Installer"
- **Error text:** `Cannot save — schema change required. Job model has no installer-tracking field; add installerId/trimVendorId (or a meta JSONB column) before wiring this to PATCH.`
- **Impact:** Raw developer error message shown to all users. PMs cannot assign installers to jobs. This is stage 9 (Installing) functionality — the exact stage where installer assignment matters most.
- **Fix:** Add `installerId` field to Job model (Prisma migration), wire the PATCH endpoint, remove the error text.

### BLOCKER-3: Material Calendar Empty
- **Location:** `/ops/material-calendar`
- **Impact:** Shows 0 jobs for current week (Apr 27 – May 3) despite 367 active jobs and 225 overdue milestones. PM opens this Monday morning to plan their week and sees "Nothing scheduled." The calendar depends on delivery dates being set on jobs, but either dates aren't propagating or the query is wrong.
- **Fix:** Audit the calendar query — it should pull from `Job.scheduledDate` or `Job.nextMilestone`. Cross-reference with jobs that have dates in PM Command Center.

### BLOCKER-4: Hyphen Sync 401 Auth Failure
- **Location:** Job detail → Documents tab → "Hyphen — SupplyPro Sync"
- **Error:** `Hyphen sync unavailable. Sync request failed (401).`
- **Impact:** PMs cannot pull plans, red-lines, change orders, or schedule events from Brookfield's portal. This affects every Brookfield job. Known issue (CLAUDE.md documents 0/80 linked).
- **Fix:** Re-authenticate the Hyphen API integration. Check stored credentials/tokens.

### BLOCKER-5: Department Portals — 404
- **Location:** `/ops/portals`
- **Impact:** The sidebar has "Department Portals" as a nav section but the page doesn't exist. PMs have no role-specific filtered view — they see the same admin dashboard as the owner.
- **Fix:** Build the portals page, or at minimum implement role-based view filtering on existing pages.

## Findings — Major Issues

### MAJOR-1: "Good evening, there" — Missing User Name
- **Location:** `/ops/my-day`
- **Impact:** Greeting doesn't include the logged-in user's name. Minor but signals the personalization layer isn't connected.
- **Fix:** Pull `user.name` or `staff.name` from the session and inject into the greeting.

### MAJOR-2: My Day — No Role Filtering
- **Location:** `/ops/my-day`
- **Impact:** A PM sees company-wide items: "2 deals inactive 14+ days" (sales), "9 builder applications to approve" (admin), "246 orders pending review" (ops-wide). Brittney doesn't need any of this — she needs her jobs, her deliveries, her builders' overdue items.
- **Fix:** Filter My Day tasks by `assignedPmId` or `role`. Show: "Your overdue jobs (X)", "Your deliveries today (X)", "Your material shorts (X)".

### MAJOR-3: Job Pipeline Cards — No Address or Job ID
- **Location:** `/ops/jobs` (Board view)
- **Impact:** Cards show only builder name, PM, and scheduled date. No job ID (JOB-2026-*), no address, no community, no lot. When Brittney has 6 Toll Brothers cards in "New," she literally cannot tell them apart.
- **Fix:** Add job ID and community/lot to the card display.

### MAJOR-4: Orders List — Rows Not Clickable
- **Location:** `/ops/orders`
- **Impact:** Even if order detail existed, the rows have no click handler. No hover state, no cursor change. PM can only look at the list — never drill into an order.
- **Fix:** Add `onClick` handler to rows, navigate to order detail page.

### MAJOR-5: $0.00 Orders in Production
- **Location:** `/ops/orders`
- **Impact:** Multiple orders show $0.00 total (SO-004177, SO-004176, SO-004171, SO-004169, SO-004167). Either pricing isn't set on these products or the total calculation is broken. 
- **Fix:** Audit orders with $0.00 total — check whether line items have unit prices. Cross-reference with Product Catalog pricing.

### MAJOR-6: All PO Recommendations $0.00
- **Location:** `/ops/orders/ai-processing`
- **Impact:** All 8 AI-recommended POs have $0.00 estimated totals. Every item price is $0.00. The AI Order Center looks impressive but is operationally useless until product pricing is populated.
- **Fix:** Populate product cost prices in the Product Catalog. The AI recommendations pull from product pricing — without it, everything zeros out.

### MAJOR-7: Placeholder Emails from Bolt Migration
- **Location:** Job detail → Builder card
- **Example:** `tollbrothers@placeholder.bolt`
- **Impact:** Fake emails visible to all users. These were placeholders during the ECI Bolt data migration and were never replaced with real contact info.
- **Fix:** Run a data cleanup script to replace all `@placeholder.bolt` emails with actual builder contacts. Flag any that can't be resolved.

### MAJOR-8: JOB-SCRUB and JOB-BOLT Data in Production
- **Location:** PM Command Center, Job Pipeline
- **Impact:** 12+ `JOB-SCRUB-2026-*` entries (AGD Homes, all $0, all CREATED) and old `JOB-BOLT-*` entries (200-400 days overdue) polluting active job lists. These inflate the 367 active jobs count and make the 225 overdue count meaningless.
- **Fix:** Archive or delete JOB-SCRUB entries. Review JOB-BOLT entries — either close out completed legacy jobs or mark them as legacy/archived.

## Findings — Enhancement Opportunities

### ENHANCE-1: Allocation Tab — Add Action Buttons
- **Current:** PM sees "2 line(s) short today. Consider a PO or reallocation." but has no way to act.
- **Need:** "Create PO" button (pre-fills with short items), "Reallocate" button, "Notify Warehouse" button.

### ENHANCE-2: Job Detail — No Inline Notes/Comments
- **Current:** No way for a PM to leave notes on a job, communicate with warehouse, or log decisions.
- **Need:** Activity feed / notes section where PM can type "Called builder, confirmed delivery for Thursday" and it gets timestamped.

### ENHANCE-3: Job Detail — No Material Shortage Detail
- **Current:** PM Command Center shows "3 SHORT" badge but the job Overview tab doesn't show which materials are short. User has to click to Allocation tab.
- **Need:** Surface the short items on the Overview tab or add a visual alert.

### ENHANCE-4: My Day — Add Calendar/Schedule View
- **Current:** My Day shows task counts only. No view of today's deliveries, installations, or meetings.
- **Need:** Timeline view: 7AM delivery at Lot 16 Lone Star Ranch, 10AM install at Canyon Ridge, 2PM builder meeting.

### ENHANCE-5: Orders List — Add Address/Community Column
- **Current:** Orders show: Order #, Status, Builder, Total, Date. No location info.
- **Need:** Add a Community / Address column so PMs can identify orders without clicking into them.

### ENHANCE-6: Job Detail — Installation Records
- **Current:** Job at INSTALLING stage shows "INSTALLATIONS (0) — No installations scheduled." The stage and the data don't match.
- **Need:** When a job advances to Installing, auto-create an installation record. Link to the assigned installer and crew.

### ENHANCE-7: Generate Standup — Test the Feature
- **Current:** PM Command Center has "Generate standup" buttons per PM. Didn't test during this audit.
- **Need:** Verify this generates a useful daily standup summary for the PM. This could be the single most valuable PM feature if it works well.

### ENHANCE-8: PM-Filtered Dashboard
- **Current:** PMs see the same admin dashboard as the owner.
- **Need:** When a PM logs in, the dashboard should show: my active jobs (count + list), my deliveries today/this week, my overdue items, my material shorts, my builders' payment status. No sales pipeline, no company-wide finance, no admin tools.

## Missing PM Tools & Automations

### Tools That Don't Exist Yet
1. **PM Dashboard** — role-filtered view showing only my jobs, my builders, my deliveries
2. **Builder Communication Hub** — send/receive messages with builder contacts from within a job
3. **Daily Delivery Schedule** — calendar view of today's deliveries with addresses, crews, time slots
4. **Material Shortage Resolver** — one-click PO creation for backordered items from the Allocation tab
5. **Change Order Workflow** — create, price, get approval, apply to order (currently buttons exist but flow is incomplete)
6. **Invoice Generator** — create invoice from completed job, auto-populate line items from order
7. **Photo Documentation** — upload job site photos tied to delivery/installation milestones

### Automations That Should Exist
1. **Auto-advance job stage** — when all materials are picked, auto-move to STAGED. When delivery is signed off, auto-move to DELIVERED.
2. **Overdue milestone alerts** — email/notification to PM when a job passes its next milestone date
3. **Material short notification** — when a reserved item goes to BACKORDERED, notify the PM
4. **Delivery confirmation email** — auto-send to builder contact when delivery is completed
5. **PM morning briefing** — auto-generate daily email: your deliveries today, overdue items, material shorts, builder payments due
6. **Pulte account lockdown** — prevent new orders/jobs from lost accounts (Pulte should not have new jobs dated 4/6, 4/8, 4/21)
7. **Installation record auto-create** — when job advances to INSTALLING, create installation record and prompt for installer assignment
