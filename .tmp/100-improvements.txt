1. SECURITY &amp; AUTH (12 items)
SEC-1 [P0]: JWT fallback allows dev-secret in production edge case
  Path: src/middleware.ts + src/lib/staff-auth.ts
  Fix: JWT_SECRET falls back to 'dev-secret-change-in-production'. While there's a runtime throw in production, the fallback line still exists. Remove the fallback entirely — crash if unset.
SEC-2 [P0]: BPW sync cron has zero auth check
  Path: src/app/api/cron/bpw-sync/route.ts
  Fix: Only cron route without CRON_SECRET validation. Even as a 410 stub, add the auth guard for consistency.
SEC-3 [P1]: Only 32 Sentry usages across 453 API routes
  Path: Codebase-wide
  Fix: Most catch blocks log to console.log (54 instances in /api alone) without reporting to Sentry. Add captureException in every catch.
SEC-4 [P1]: No CSRF protection on state-mutating ops routes
  Path: src/middleware.ts
  Fix: Middleware has CSRF comment at line 275 but only skips logging endpoints. No actual CSRF token validation on POST/PATCH/DELETE for ops routes.
SEC-5 [P1]: Admin routes check session existence only, not role
  Path: src/middleware.ts line 681
  Fix: Comment says 'ADMIN role enforced in middleware' but verify the actual check gates on role=ADMIN, not just valid-session.
SEC-6 [P1]: Hyphen OAuth credentials stored in plaintext
  Path: src/lib/hyphen/scraper.ts + integration config
  Fix: Hyphen login uses hardcoded or DB-stored plaintext credentials. Encrypt at rest or use env vars exclusively.
SEC-7 [P2]: No rate limiting on auth endpoints
  Path: src/app/api/auth/login/route.ts, /signup, /reset-password
  Fix: Auth routes have no rate limiter. Add Upstash ratelimit (already in stack) to login/signup/reset with IP-based sliding window.
SEC-8 [P2]: Password reset tokens lack explicit expiration check
  Path: src/app/api/auth/reset-password/route.ts
  Fix: Verify that reset tokens have TTL enforcement. If using DB-stored tokens, add expiresAt column + check.
SEC-9 [P2]: File uploads have no size limit enforcement
  Path: src/app/api/ops/ (various upload routes)
  Fix: No maxFileSize or Content-Length validation on blueprint/document upload endpoints. Add 25MB cap.
SEC-10 [P2]: No Content-Security-Policy header
  Path: src/middleware.ts
  Fix: Add CSP header (script-src, style-src, img-src) to prevent XSS. Vercel supports this in middleware.
SEC-11 [P2]: Agent SMS webhook returns 501 with no auth
  Path: src/app/api/agent/sms/route.ts
  Fix: Endpoint is publicly accessible and returns 501. Either add Twilio signature validation or remove the route.
SEC-12 [P2]: NUC integration endpoints lack auth
  Path: src/app/api/integrations/nuc/health/route.ts, .../query/route.ts
  Fix: No try/catch, no auth check. These should require CRON_SECRET or staff session.
2. DATA INTEGRITY &amp; SCHEMA (15 items)
DATA-1 [P0]: Zero soft-delete pattern anywhere in 239 models
  Path: prisma/schema.prisma
  Fix: No deletedAt or isDeleted column on any model. Permanent deletes break audit trails. Add deletedAt DateTime? to Builder, Order, Quote, Invoice, Job, Product at minimum.
DATA-2 [P0]: onDelete: Restrict on Order.builderId prevents account cleanup
  Path: prisma/schema.prisma line 584
  Fix: Cannot delete or archive a builder without first reassigning/deleting all orders. Add soft-delete or change to SetNull with nullable builderId.
DATA-3 [P0]: onDelete: Restrict on QuoteItem.productId blocks product retirement
  Path: prisma/schema.prisma line 558
  Fix: Can't deactivate a product that's ever been quoted. Change to SetNull + add productSnapshot JSON field for historical record.
DATA-4 [P1]: onDelete: Restrict on OrderItem.productId blocks product deletion
  Path: prisma/schema.prisma line 656
  Fix: Same issue as DATA-3 for order items.
DATA-5 [P1]: Quote.takeoffId unique constraint + cascade creates orphan risk
  Path: prisma/schema.prisma
  Fix: If takeoff cascades on blueprint delete, the Quote loses its takeoff but the Quote itself persists in a broken state.
DATA-6 [P1]: No index on Builder.status — filters scan entire table
  Path: prisma/schema.prisma
  Fix: Builder queries filter by status (ACTIVE/PENDING/SUSPENDED) without index. Add @@index([status]).
DATA-7 [P1]: No index on Order.status — most common filter
  Path: prisma/schema.prisma
  Fix: Order list pages always filter/sort by status. Add @@index([status, createdAt]).
DATA-8 [P1]: No index on Job.phase — ops queries are slow
  Path: prisma/schema.prisma
  Fix: Manufacturing/delivery pages filter jobs by phase. Add @@index([phase, scheduledDate]).
DATA-9 [P1]: No index on Invoice.dueDate — collections queries
  Path: prisma/schema.prisma
  Fix: Collections cycle queries invoices by dueDate ranges. Add @@index([status, dueDate]).
DATA-10 [P2]: No index on Delivery.status + date
  Path: prisma/schema.prisma
  Fix: Today's deliveries page and dispatch logic filter by status + scheduledDate. Add composite index.
DATA-11 [P2]: 667 indexes for 239 models — but key operational fields missed
  Path: prisma/schema.prisma
  Fix: Existing indexes cover relations (FK) but not the status/date/phase fields actually used in WHERE clauses.
DATA-12 [P2]: Legacy models still in schema (Bolt*, Bpw*, QbCustomer*)
  Path: prisma/schema.prisma
  Fix: ECI Bolt sync, Pulte BPW, and QB models are dead. Mark deprecated or remove to reduce schema noise.
DATA-13 [P2]: No database-level constraint on Order total vs line items
  Path: prisma/schema.prisma + recompute-order-totals cron
  Fix: Order.total can drift from sum(OrderItem.total). The cron fixes it but add a CHECK constraint or trigger.
DATA-14 [P2]: Staff assignment uses SetNull — pages expect non-null
  Path: prisma/schema.prisma (various assigneeId fields)
  Fix: When staff is deleted, assigned orders/jobs become null but UI doesn't handle the null state gracefully.
DATA-15 [P2]: No unique constraint on Product.sku
  Path: prisma/schema.prisma
  Fix: Verify sku is unique. Duplicate SKUs from InFlow imports create confusion. Add @@unique([sku]) if not present.
