## Recommendations (post-launch)

1. **Bring the 27 raw-SQL-only tables into Prisma via `prisma db pull`** (introspection), review the generated models, rename relations where needed, then commit. Highest priority: the ingest tables that survive reboots (`BankEntry`, `CalendarEvent`, `LegalNote`, `VendorNote`, `Sop`, `ArHistorySnapshot`, `*PlanBom`, `PulteHistoricalBackcharge`).

2. **Kill the "self-heal create" pattern for already-modeled tables.** Files like `src/app/api/ops/sales/migrate/route.ts`, `ops/migrate-all/route.ts`, `ops/migrate-nfc/route.ts`, `ops/migrate/platform-upgrade/route.ts`, `ops/migrate-outreach/route.ts` each CREATE a dozen+ tables that already exist in Prisma. These are legacy one-shot migration endpoints — consider marking them deprecated or deleting them once live DB is confirmed to have everything.

3. **Prefix typo — `Bwp*` vs `Bpw*`.** The codebase uses **both** `BwpStagingData` (intake), `BwpFieldPO`/`BwpFieldPOLine`/`BwpInvoice`/`BwpCheck`/`BwpCommunity`/`BwpBackcharge` (modeled), **AND** `BpwStagingData`/`BpwFieldPO`/`BpwInvoice`/`BpwCheck`/`BpwJobDetail`/`BpwCommunity` (raw-CREATE in some routes). There are parallel tables with near-identical schemas — confirm which is live and consolidate. Sample evidence:
   - `scripts/import-bwp-pulte.mjs` creates `BwpFieldPO`, `BwpInvoice`, `BwpCheck`, `BwpContact`, `BwpBackcharge` (matches Prisma).
   - `src/app/api/ops/import-bpw/route.ts` + `…/process/route.ts` create `BpwFieldPO`, `BpwInvoice`, `BpwCheck`, `BpwCommunity`, `BpwJobDetail` (no Prisma models with `Bpw` prefix — these are also drift).

4. **Formalize the pattern going forward.** Use `prisma migrate dev` / `prisma migrate deploy` for every new table. Reserve raw `CREATE TABLE IF NOT EXISTS` for (a) idempotent self-healing of truly optional telemetry tables (`ClientError`, `ServerError`, `SlowQueryLog`, `CronRun`, `EmailSendLog`) where losing the DDL is acceptable, or (b) scripts that must run against arbitrary environments. Everything else should live in migrations.

5. **Delete or gate the `/api/ops/migrate-*/route.ts` endpoints** before launch — anyone with the cookie can hit them and issue DDL. At minimum, require a fresh admin-secret header and log every invocation to `AuditLog`.

6. **`AgentWorkflow` is fully off-Prisma** but is read and written by real routes. Either model it or consolidate with `AgentTask` / `AgentSession` which are modeled and have overlapping intent.

7. **Review `Bpw*` drift tables' schemas** (`BpwStagingData`, `BpwCommunity`, `BpwJobDetail`, `BpwFieldPO`, `BpwInvoice`, `BpwCheck`) and compare to their `Bwp*` Prisma siblings — the safe move is to fold their data into the modeled `Bwp*` tables and drop the `Bpw*` variants entirely.
