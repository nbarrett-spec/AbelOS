PROJECT OVERVIEW
================
A mobile-first crew management portal for delivery and installation teams.
Built with Next.js 14 App Router, TypeScript, Tailwind CSS, and Prisma.

Core Features:
- Daily schedule management
- Delivery workflow tracking (6 steps)
- Installation workflow tracking (3 steps)
- Route optimization view
- Crew member management
- Status persistence and validation
- Mobile-optimized interface

================================================================================
WORKFLOWS
================================================================================

DELIVERY (6 steps):
  1. SCHEDULED      → 2. LOADING     → 3. IN_TRANSIT
  4. ARRIVED        → 5. UNLOADING   → 6. COMPLETE
  
  Requirements:
  - Signature required for completion
  - Timestamp captured per step
  - No step skipping allowed

INSTALLATION (3 steps):
  1. SCHEDULED      → 2. IN_PROGRESS → 3. COMPLETE
  
  Requirements:
  - QC pass required for completion
  - Punch list optional
  - Timestamp captured per step

================================================================================
DATABASE MODELS
================================================================================

Key relationships:
  Staff (id, name, email, phone, role)
    ↓ via CrewMember
  Crew (id, name, crewType, vehiclePlate)
    ↓
  ScheduleEntry (jobId, crewId, type, date, time, status)
    ↓
  Job (id, jobNumber, builderName, address, status)
    ↓
  Delivery/Installation (jobId, crewId, status, timestamps)
  MaterialPick (jobId, sku, description, quantity)
