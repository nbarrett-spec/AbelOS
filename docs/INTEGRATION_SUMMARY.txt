DATABASE SCHEMA
================================================================================

Auto-created table: SupplierPriceUpdate
- Stores supplier price changes pending review
- 20 columns covering cost, price, margin, match quality, audit trail
- 4 indexes for efficient querying
- Status workflow: PENDING → APPROVED/REJECTED

Related tables (already exist in Abel system):
- Product (linked via productId)
- SyncLog (audit trail)

================================================================================
KEY FEATURES
================================================================================

✅ CSV PARSING
   • Flexible column detection (case-insensitive, partial matches)
   • Handles quoted fields, escaped quotes
   • Auto-converts currency ($) and numbers (commas)
   • Supports multiple Boise Cascade formats

✅ SKU MATCHING (3-tier strategy)
   1. Exact: SKU = SKU (Confidence: 100%)
   2. Fuzzy: PostgreSQL SIMILARITY() >50% (Confidence: 40-95%)
   3. Partial: First significant word match (Confidence: 40%)
   ✗ No match: Reported for manual review

✅ PRICE CALCULATIONS
   • Cost change tracking (absolute $ and %)
   • Margin impact analysis
   • Automatic price adjustment to maintain minMargin
   • Suggested pricing based on margin thresholds

✅ BATCH WORKFLOW
   Upload CSV → Parse & Match → Calculate Changes → Store PENDING
        ↓
   Review (with alerts for low margins)
        ↓
   Approve / Reject / Approve-All
        ↓
   Update Product.cost + Log to SyncLog

✅ SAFETY FEATURES
   • Price alerts for items below margin threshold
   • Cost increases flagged for review
   • Audit trail (staff member, timestamp)
   • Batch grouping for tracking
   • Match confidence scoring

✅ ANALYTICS & REPORTING
   • Import history with batch statistics
   • Cost change summary (avg, min, max, total impact)
   • Product updates by category
   • Approval rates and trends
   • Sync logs for compliance

API ENDPOINTS
================================================================================

1. GET /api/ops/integrations/supplier-pricing
   └─ Overview, pending counts, cost changes, price alerts, history

2. POST /api/ops/integrations/supplier-pricing
   └─ Upload CSV, parse, match SKUs, store as PENDING updates

CSV FORMAT SUPPORTED
================================================================================

Boise Cascade standard:
Item Number,Description,UOM,List Price,Net Price,Effective Date

Column name variations automatically detected:
- SKU: Item Number, Item #, SKU, item, product code
- Description: Description, Product Name, Product, name
- Cost: Net Price, Dealer Price, Our Price, Cost, Your Cost
- Price: List Price, MSRP
- UOM: UOM, Unit of Measure, Unit

Handles:
- Quoted fields
- Dollar signs ($)
- Comma-separated numbers (1,234.50)
- Empty rows
- Multiple formats in same CSV

================================================================================
MARGIN PROTECTION LOGIC
================================================================================

Example: Product with minMargin = 25%

Scenario: Cost increases from $100 → $120
- Current price: $300
- Current margin: ($300-$100)/$300 = 66.7%
- At new cost: ($300-$120)/$300 = 60% ✓ (still above 25%)

Scenario: Cost increases from $150 → $180
- Current price: $200
- Current margin: ($200-$150)/$200 = 25% (exactly at threshold)
- At new cost: ($200-$180)/$200 = 10% ✗ (BELOW threshold)
- Suggested price: $180 / (1 - 0.25) = $240
- This maintains 25% margin

System automatically:
1. Calculates margin impact
2. Flags items below threshold
3. Suggests price adjustments
4. Allows manual override on approval
